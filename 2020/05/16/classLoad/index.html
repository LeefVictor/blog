对JVM如何加载类， 以及加载类过程中做了什么操作，做一个简单的笔记。

类的加载

即.class文件的加载, 在jvm需要使用到该类时对其编译文件进行装置的过程. (并发一定是在使用到才加载, jvm可以通过某些判断逻辑, 当其即将被使用时,提前进行加载). 类的结构主要加载在jvm的方法区.
加载方式:

  • 本地中加载
  • 网络中下载
  • jar包中加载
  • .java源文件动态编译

类的生命周期

load-->verificate-->prepare-->resolute-->init-->usage-->unload

加载

第一个阶段，加载.class二进制文件， 将其静态结构存入方法区， 同时，在heap中也存入一个 Class类对象。

验证

校验文件格式， 字节码信息， 字节码规范等等是否符合当前jvm虚拟机的规范和要求。此阶段不是必须的，它对程序运行期没有影响，当确认所使用文件无错，那么可以考虑采用 -Xverifynone参数来关闭大部分的类验证措施，以缩短虚拟机类加载的时间。

准备

为类变量（final static, static修饰的变量）分配内存并根据其类型设置初始值。

  • final static 和 final修饰的变量， 在此阶段就会被初始化为其赋予的值
  • static修饰的变量， 则在此阶段初始化为其类型的默认值， 比如
    private static int a = 9; ==> 在此处a是初始化为0， 赋值为9则是在另一个阶段

上述的变量都是随类结构存于jvm方法区中

初始化

对类的静态变量进行初始化， 和对类对象的初始化。
静态变量就很好解析了， 就是上面准备阶段的第二点， 对a变量赋值为9。
同时对类实例进行创建，和对实例变量进行初始化。

此阶段只有主动使用类是才会进行。即在：

  • 创建实例 new xxx()
  • 访问类静态变量 XXX.a 或调用其静态方法
  • 反射
  • 初始化其子类，父类也会初始化

卸载

无法手动卸载指定类， 只有当执行了System.exit或者程序因其他异常终止时， 才会解析器生命周期。
